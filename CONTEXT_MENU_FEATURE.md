# 消息右键菜单功能说明

## 功能概述

已成功为AI对话窗口的消息气泡添加了完整的右键上下文菜单，提供更丰富的消息操作功能。

## 实现的功能

### 📋 复制文本
- 复制消息的原始文本到剪贴板
- 保留格式和换行
- 显示"已复制到剪贴板"提示

### 📝 编辑消息（仅用户消息）
- 弹出编辑对话框
- 修改消息内容
- 保存后删除后续所有消息并重新发送
- 同步更新历史记录

### 🔄 重新生成（仅AI消息）
- 删除当前AI回复及后续消息
- 自动找到对应的用户消息
- 重新发送给AI生成新回复
- 适用于对AI回复不满意时

### 💾 导出为文件
- 支持导出为 .txt 或 .md 格式
- 包含消息元数据（角色、时间）
- 自定义文件名和保存路径
- 显示导出成功提示

### ▶ 执行此SQL（仅SQL消息）
- 智能识别SQL内容
- 从代码块中提取SQL
- 执行前显示确认对话框
- 展示执行结果（列名、行数、前10行数据）
- 错误处理和友好提示

### 🗑️ 删除此消息
- 二次确认防止误删
- 同步删除UI和历史记录
- 保持对话连续性

## 使用场景示例

### 场景1：修正问题后重新提问
1. 发现刚才的问题描述不够清晰
2. 右键用户消息 → 编辑消息
3. 修改问题描述
4. 点击"保存并重新发送"
5. AI基于新问题重新回答

### 场景2：对AI回复不满意
1. AI的回答不够详细或有误
2. 右键AI消息 → 重新生成
3. AI基于相同问题生成新回复
4. 可以多次尝试直到满意

### 场景3：快速执行SQL
1. AI生成了SQL查询语句
2. 右键SQL消息 → 执行此SQL
3. 确认执行
4. 查看查询结果

### 场景4：保存重要对话
1. 遇到有价值的AI建议
2. 右键消息 → 导出为文件
3. 选择保存位置和格式
4. 以后可以随时查阅

## 已知限制

1. **编辑消息**会删除该消息之后的所有内容（包括AI回复）
   - 原因：保持对话逻辑一致性
   - 改进方向：未来可支持插入式编辑

2. **重新生成**需要网络正常
   - 失败时会显示错误提示
   - 不会丢失原有消息

3. **SQL执行**依赖数据库连接
   - 需要先配置有效的数据库连接
   - 只读操作更安全
