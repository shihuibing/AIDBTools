# 键盘快捷键功能说明

## 功能概述

已成功为AI对话窗口添加了丰富的键盘快捷键，大幅提升操作效率，让用户可以像专业开发者一样快速操作。

## 快捷键列表

### 📝 输入区快捷键

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| **Enter** | 发送消息 | 快速发送当前输入 |
| **Shift+Enter** | 换行 | 在输入框中插入新行 |
| **Ctrl+K** | 清空输入 | 清除当前输入内容 |
| **Ctrl+L** | 聚焦输入 | 快速定位到输入框并全选 |
| **@** | 选择数据库 | 弹出数据库选择菜单 |

### ⬆️⬇️ 历史导航

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| **Alt+↑** | 上一条消息 | 浏览之前发送的用户消息 |
| **Alt+↓** | 下一条消息 | 向前导航，最后恢复草稿 |

### 🎯 全局快捷键

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| **Esc** | 停止生成 | 中断当前的AI请求 |
| **Ctrl+Shift+H** | 切换历史面板 | 显示/隐藏左侧历史会话面板 |
| **Ctrl+F** | 搜索（预留） | 未来将支持对话内搜索 |

## 详细功能说明

### 1. Enter / Shift+Enter

**Enter - 发送消息**
```
用户输入: "查询所有用户"
按下: Enter
结果: 消息立即发送，输入框清空
```

**Shift+Enter - 换行**
```
用户输入: "第一行"
按下: Shift+Enter
结果: 光标移到下一行，可以继续输入
```

**使用场景**:
- 编写多行SQL语句
- 格式化代码片段
- 撰写结构化问题

---

### 2. Ctrl+K - 清空输入

```
当前输入: "这是一段很长的文字..."
按下: Ctrl+K
结果: 输入框立即清空，焦点保持不变
```

**使用场景**:
- 写错了想重来
- 复制粘贴后想清除
- 快速放弃当前草稿

---

### 3. Ctrl+L - 聚焦输入

```
当前焦点: 任意位置
按下: Ctrl+L
结果: 焦点跳到输入框，所有内容被选中
```

**使用场景**:
- 从其他应用切换回来
- 快速替换整个输入
- 鼠标不便使用时

---

### 4. Alt+↑/↓ - 历史导航

这是最实用的功能之一，类似终端的历史命令导航！

**工作流程**:
```
1. 用户发送了3条消息:
   - "查询用户表结构"
   - "优化这个SQL"
   - "解释执行计划"

2. 当前输入框为空

3. 按下 Alt+↑
   → 显示: "解释执行计划"
   → 状态提示: "历史消息 3/3"

4. 再按 Alt+↑
   → 显示: "优化这个SQL"
   → 状态提示: "历史消息 2/3"

5. 再按 Alt+↑
   → 显示: "查询用户表结构"
   → 状态提示: "历史消息 1/3"

6. 按 Alt+↓
   → 显示: "优化这个SQL"
   → 状态提示: "历史消息 2/3"

7. 继续按 Alt+↓ 超出范围
   → 恢复之前保存的草稿内容
```

**智能特性**:
- ✅ 首次按Alt+↑时自动保存当前草稿
- ✅ 导航超出范围时恢复草稿
- ✅ 发送消息后自动重置导航状态
- ✅ 显示当前位置提示（X/Y）

**使用场景**:
- 修改之前的问题重新发送
- 参考之前的提问方式
- 快速重用常用查询
- 找回误删的草稿

---

### 5. Esc - 停止生成

```
状态: AI正在生成回复
按下: Esc
结果: 
  - 立即停止AI请求
  - 隐藏加载指示器
  - 恢复发送按钮
  - 显示"已停止"提示
```

**使用场景**:
- AI回答方向不对
- 问题描述有误需要修正
- 长时间无响应想取消

---

### 6. Ctrl+Shift+H - 切换历史面板

```
状态: 历史面板隐藏
按下: Ctrl+Shift+H
结果: 
  - 左侧历史面板滑出
  - 显示最近会话列表
  - 按钮图标变化

再按一次:
  - 历史面板收起
  - 聊天区域扩展
```

**使用场景**:
- 快速查看历史会话
- 最大化聊天空间
- 键盘党专属操作

---

### 7. @ - 数据库选择

```
输入框中输入: @
结果: 
  - 弹出数据库选择菜单
  - 显示所有可用数据库
  - 点击即可插入@db:xxx标签
```

**使用场景**:
- 切换数据库上下文
- 引用特定数据库
- 多库对比查询

## 技术实现

### 核心方法

#### 1. eventFilter - 事件拦截
```python
def eventFilter(self, obj, event):
    """拦截输入框的键盘事件"""
    if obj is self.input_box and event.type() == QEvent.Type.KeyPress:
        # 检测各种快捷键组合
        if event.key() == Qt.Key.Key_Return:
            if not (event.modifiers() & Qt.KeyboardModifier.ShiftModifier):
                self._on_send()
                return True
        # ... 其他快捷键
    return super().eventFilter(obj, event)
```

#### 2. _navigate_history - 历史导航
```python
def _navigate_history(self, direction: int):
    """导航历史记录"""
    # 获取所有用户消息
    user_messages = [msg for msg in history if msg.get("role") == "user"]
    
    # 首次导航时保存草稿
    if self._history_navigation_index == -1:
        self._temp_input_buffer = self.input_box.toPlainText()
        self._history_navigation_index = len(user_messages)
    
    # 计算新索引并显示
    new_index = self._history_navigation_index + direction
    if 0 <= new_index < len(user_messages):
        self._history_navigation_index = new_index
        self.input_box.setPlainText(user_messages[new_index]["content"])
```

### 状态管理

```python
# 历史导航状态
self._history_navigation_index: int = -1  # 当前导航位置
self._temp_input_buffer: str = ""          # 临时草稿缓冲
```

### 生命周期

1. **初始状态**: `_history_navigation_index = -1`
2. **首次Alt+Up**: 保存草稿，索引设为消息总数
3. **导航中**: 索引在 0 ~ 消息数-1 之间变化
4. **发送消息**: 调用 `_reset_history_navigation()` 重置
5. **超出范围**: 恢复草稿，索引重置为-1

1. **Esc** - 全局最高优先级
2. **输入框快捷键** - 当焦点在输入框时
3. **全局快捷键** - 任何位置都有效

两种方式达到相同目的！

```

  1. 输入 SELECT...
  2. Shift+Enter 换行
  3. 输入 FROM...
  4. Shift+Enter 换行
  5. 输入 WHERE...
  6. Enter 发送
```

## 自定义快捷键

如果需要添加新的快捷键，在 `eventFilter` 方法中添加：

```python
# 示例: 添加 Ctrl+E 导出对话
if (event.key() == Qt.Key.Key_E and 
    (event.modifiers() & Qt.KeyboardModifier.ControlModifier)):
    self._export_conversation()
    return True
```
